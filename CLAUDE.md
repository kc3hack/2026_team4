# Claude Code 設定（Pikumei）

## プロジェクト概要
- iOSアプリ（Swift / SwiftUI）
- アーキテクチャ：MVVM
- ハッカソン向け（完成優先）

## コーディング方針
- SwiftUI は View にのみ書く
- ViewModel は状態管理と処理呼び出しのみ
- Model は struct / enum のみ
- ロジック・CoreML・Camera は ViewModel 配下に実装する

## 実装ルール
- 新規ファイル作成時は、必ず配置先フォルダを明示する
- 既存構成を勝手に変更しない
- 不要な抽象化・レイヤー分割は行わない
- 完成度より分かりやすさを優先する

## 命名規則
- `View/Component/` 配下のコンポーネントは `〇〇Component` と命名する

## コードスタイル
- SwiftUI はシンプルに書く
- 1ファイルに責務を詰め込みすぎない
- コメントは「なぜ」を書く

## 禁止事項
- Clean Architecture への拡張
- Repository / UseCase の追加
- 勝手な依存関係の導入

## 出力ルール
- ファイル単位でコードを提示する
- README や設計を勝手に変更しない
- 不明点があれば実装前に質問する

## Git 運用ルール（重要）
- コミット前に **必ず y / n で確認を取る**
- ユーザーの明示的な指示なしに commit / push を行わない
- コミットの単位は小さく保つ（1コミット1変更）
- コミットメッセージは **日本語** で記述する
